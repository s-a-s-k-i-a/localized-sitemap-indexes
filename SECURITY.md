# Security Policy

## Supported versions

Before a `1.0.0` release, security fixes are handled on the latest tagged release and the current development branch.

## Reporting a vulnerability

Do not open public GitHub issues for security problems.

Use GitHub private vulnerability reporting for this repository once it is enabled. If that is not available, contact the maintainer through an existing private project channel and include:

- affected plugin version
- affected WordPress, PHP, and dependency versions
- clear reproduction steps
- expected impact
