Adds language-specific XML sitemap indexes for TranslatePress while mirroring Rank Math sitemap visibility rules.

== Description ==

Localized Sitemap Indexes is for WordPress sites that use:

- TranslatePress for language directories
- Rank Math SEO for XML sitemaps
- optionally NitroPack for cache warmup

The plugin adds separate sitemap index files per published TranslatePress language, for example:

- `sitemap_index_en.xml`
- `sitemap_index_fr.xml`
- `sitemap_index_de.xml`

Each language-specific index points to language-specific child sitemaps such as:

- `sitemap_en_post_1.xml`
- `sitemap_fr_product_1.xml`

This lets you keep Rank Math's main `sitemap_index.xml` intact while exposing cleaner multilingual discovery paths for crawlers and separate sitemap targets for cache warmup systems.

The plugin reads its language list from TranslatePress and its object visibility from Rank Math. If you disable a post type or taxonomy in Rank Math sitemap settings, it is also excluded from the language-specific indexes generated by this plugin.

Optional NitroPack support:

- exposes `nitro-warmup-sitemap.xml`
- provides `wp localized-sitemaps sync-nitro`

== Features ==

- Dynamic language-specific sitemap indexes based on TranslatePress published languages
- Rank Math post type and taxonomy sitemap visibility respected
- Rank Math global noindex and item-level `rank_math_robots` exclusions respected
- Rank Math `items_per_page` respected
- `robots.txt` advertisement for language-specific sitemap indexes
- optional separate NitroPack warmup sitemap index
- cache invalidation on language, sitemap, post, and term changes
- `wp localized-sitemaps list-indexes` for operator-facing endpoint checks

== Requirements ==

- WordPress
- TranslatePress
- Rank Math SEO

NitroPack is optional and only needed if you want to use the dedicated warmup index and WP-CLI warmup sync command.

== Installation ==

1. Upload the plugin folder to `/wp-content/plugins/`.
2. Activate the plugin.
3. Make sure TranslatePress and Rank Math SEO are active.
4. Open `robots.txt` and verify that your language-specific sitemap indexes are listed.

If you use NitroPack and want it to follow the dedicated warmup index:

1. Run `wp localized-sitemaps sync-nitro`
2. Start a fresh NitroPack warmup

To inspect which top-level sitemap endpoints are currently available:

1. Run `wp localized-sitemaps list-indexes`

== Frequently Asked Questions ==

= Does this replace Rank Math's default sitemap index? =

No. Rank Math's main `sitemap_index.xml` stays untouched. This plugin adds additional language-specific indexes.

= Does this work without TranslatePress? =

No. The language model comes from TranslatePress.

= Does this work without Rank Math? =

No. The plugin is designed to mirror Rank Math sitemap visibility and sizing logic.

= Does this support hreflang sitemap annotations? =

No. This plugin focuses on language-specific sitemap indexes and URLs. It does not inject hreflang alternates into XML.

= Is NitroPack required? =

No. NitroPack support is optional.

= Does this support separate domains per language? =

Not in the current scope. The first public versions target TranslatePress directory-based language URLs.

== Limitations ==

- The first public version is focused on post type and taxonomy sitemap parity.
- It does not yet mirror every specialized Rank Math sitemap provider such as author, date, image, video, or news sitemap behavior.
- It assumes a TranslatePress URL-directory language setup rather than separate domains per language.
